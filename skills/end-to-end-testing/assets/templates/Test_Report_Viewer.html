[Environment Creation] API returns 500 error when creating Headless game environment

Reporter: Claude Code (Automated Testing)

Date Found: 2025-11-12 09:22 UTC

Module: Game Environment Management - Environment Creation API

Description

API endpoint POST /api/create_game_env returns HTTP 500 Internal Server Error when attempting to create Headless game environments. Error message: {"message":"Error creating service"}

Root Cause (Suspected)

  1. Node Label Mismatch: The x86-ng-v2 node group exists and is ACTIVE, but nodes may not have the correct node-group=x86-ng-v2 label that the pod scheduler is looking for.
  2. Node Availability: Although the node group has desiredSize=2, the nodes might not be ready or properly joined to the cluster yet.
  3. Service Account / RBAC Permissions: The backend service may lack sufficient Kubernetes permissions to create deployments/services in the target namespace.

Investigation Results

# Check for nodes with x86-ng-v2 label $ kubectl get nodes -l node-group=x86-ng-v2 No resources found # Check EKS node group status $ aws eks describe-nodegroup --cluster-name ai-bot-cluster --nodegroup-name x86-ng-v2 { "status": "ACTIVE", "scalingConfig": { "minSize": 1, "maxSize": 10, "desiredSize": 2 } }

Impact

  • User Impact: HIGH - Users cannot create any game environments, which is a core feature
  • Workflow Impact: CRITICAL - Blocks model training workflow
  • Test Coverage Impact: Blocks TC-2.1, TC-2.2, TC-3.1, TC-5.1 (4 out of 8 test cases = 50%)

Recommendations

  1. Immediate: Check backend application logs for detailed stack trace
  2. Immediate: Verify node group labels match what the API expects
  3. Immediate: Confirm Kubernetes service account has correct permissions
  4. Short-term: Improve error messages to include actionable details
  5. Short-term: Add API endpoint validation before creating Kubernetes resources
  6. Long-term: Implement pre-flight checks in UI to validate node group availability
